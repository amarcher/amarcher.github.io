JavaScript Pros & Cons

Wednesday, August 19, 2014

Why is JavaScript popular in web development?

JavaScript was developed in 1995 by Netscape for its then-popular browser: Netscape Navigator. It allowed programmers to code dynamic and interactive web-pages by providing the browsers with instructions to manipulate the web-page's content, formatting, and structure via its Document Object Model (DOM). Because it was the only languge Netscape Navigator supported, it was the de facto language-of-choice for any developer who wanted to make dynamic and interactive websites that could run on all popular browser platforms. As if it needed additional name-recognition: JavaScript also encapsulated the name "Java" within its name although it had almost no relation. The "Java" language was already popular at the time for software development.

What are the great things about JavaScript?

Aside from enabling dynamic web design, has a few other great features. Principly, JavaScript is fast and loose with data types. As a programmer this means two things: (1) you don't have to fear compile-time errors: your program rums and doesn't exit when it hits an error and (2) you don't have to do lots of rigorous type-checking (as you might in languages like Objective-C) which add lots of extra lines of code.

What are its limitations?

JavaScript isn't pefect and has some limitations. Notable among these: (1) it cannot communicate with the server (to fetch or store information from server databases) without using techniques like AJAX and (2) it cannot write information on the client save for in small packets called cookies which are prescribed strict access limitations by the browser.
